Guide

概要

Amazon EC2（Elastic Compute Cloud）は、AWSが提供するクラウドコンピューティングサービスである。 ユーザーは仮想サーバー（インスタンス）をオンデマンドで作成し、実行することができる。 物理的なサーバーを購入・管理する必要がなく、必要に応じて柔軟にコンピューティングリソースを利用できる点が大きな特徴である。

EC2の主な特徴:

  1. 仮想サーバーの提供 (インスタンス): EC2は、CPU、メモリ、ストレージなどのリソースをユーザーのニーズに応じて柔軟に選択できる仮想サーバーを提供する。これにより、物理サーバーの管理コストや導入の煩雑さを削減できる。

  2. スケーラビリティ: EC2は、リソースを自動で拡張・縮小できるスケーラブルなサービスである。需要が増減する状況に応じて、インスタンスの数や規模を柔軟に調整することが可能で、効率的なリソース運用を支援する。

  3. 多様なインスタンスタイプ: EC2では、汎用型、計算集約型、メモリ集約型、GPU搭載型など、多様なインスタンスタイプを提供している。これにより、ユーザーはアプリケーションの特定の要件に応じた最適なインスタンスを選択できる。

  4. 柔軟な料金体系: EC2は、ユーザーの利用スタイルに応じた料金プランを用意している。

  • オンデマンドインスタンス: 必要なときにのみ利用し、その分だけの料金を支払う形態。

  • リザーブドインスタンス: 1年または3年単位で契約し、長期利用により割引を受けられる。

  • スポットインスタンス: 空きリソースを利用することで、通常よりも低価格で使用できるが、予告なく利用が中断される可能性がある。

  1. セキュリティとアクセス制御: EC2では、AWS IAM（Identity and Access Management）を活用することで、アクセス権限を詳細に管理することが可能である。また、仮想プライベートクラウド（VPC）を使用してネットワークを分離し、セキュリティグループによりインスタンスへの通信を制御できる。

  2. ストレージオプション: インスタンスに直接紐づく一時的なローカルストレージであるインスタンスストアと、永続的なデータ保存が可能なAmazon EBS（Elastic Block Store）の2つの主要なストレージオプションを提供している。

  3. 地域とアベイラビリティゾーン: 世界中に分散したAWSのデータセンターを利用でき、ユーザーは最適な場所でインスタンスを実行することが可能である。これにより、低遅延や高可用性を実現する。

ユースケース:

EC2は、以下のようなさまざまなユースケースに適している。

  • ウェブアプリケーションのホスティング

  • データベースサーバーの運用

  • 大規模なデータ分析

  • 機械学習モデルのトレーニング

  • 高性能コンピューティング（HPC）など

機能

機能

説明

インスタンスタイプの選択

汎用、コンピューティング最適化、メモリ最適化、ストレージ最適化、アクセラレータ（GPU）最適化など、多様なインスタンスタイプを提供。

オートスケーリング

負荷に応じて自動的にインスタンスを増減させ、パフォーマンスとコスト効率を最適化。

Elastic Load Balancing (ELB)

トラフィックを複数のインスタンスに分散させ、システムの高可用性を実現。

Amazon EBS

永続的なブロックストレージで、スナップショットによるバックアップや復元も可能。

Amazon EC2 Auto Recovery

障害発生時にインスタンスを自動再起動し、高可用性を確保。

Elastic IP アドレス

固定IPv4アドレスをインスタンスに割り当て、動的IP変更にも対応可能。

セキュリティグループ

仮想ファイアウォールとしてインスタンスへのトラフィックを制御。

Virtual Private Cloud (VPC)

仮想ネットワークを作成し、インスタンスのネットワークを管理。

キーペアによる認証

SSHを利用したインスタンスへのアクセスを公開鍵暗号方式で管理。

CloudWatchによるモニタリング

インスタンスのリソース使用率を監視し、必要に応じてアラームやスケーリングを設定可能。

スポットインスタンス

未使用リソースを安価に利用できるが、予告なく中断される可能性あり。

リザーブドインスタンス

長期利用でインスタンスを予約し、コスト削減を実現。

Dedicated Hosts

ユーザー専用の物理サーバーを割り当て、ライセンスやコンプライアンスに対応。

ホストリソースのモニタリング

リソースの使用状況をリアルタイムで監視し、パフォーマンス管理が可能。

EC2フリート

複数のインスタンスタイプを組み合わせて、最適な価格とパフォーマンスを実現。

Elastic Fabric Adapter (EFA)

高性能コンピューティングや機械学習向けに低レイテンシのネットワークを提供。

Placement Groups

インスタンスを近くに配置し、高スループットと低レイテンシの通信を実現。

インスタンスメタデータ

インスタンス情報を取得するためのメタデータ機能。

タグ付け

リソースにカスタムタグを付け、管理や分類を容易にする。

停止・再開機能

インスタンスを停止して再開することで、データを保持したまま中断可能。

ハイバネーション

メモリの内容を保存したままインスタンスをサスペンド状態にして後で復元可能。
